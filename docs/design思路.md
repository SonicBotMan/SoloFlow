# 🎬 AI一人公司 - 产品设计思路

## 一、核心设计理念

### 1.1 产品定位

```
┌─────────────────────────────────────────────────────────────────┐
│                    AI一人公司 = 老板 + AI员工团队                  │
├─────────────────────────────────────────────────────────────────┤
│  老板（用户）←→ 老板助理（AI）←→ 员工团队（AI）                   │
│                                                                 │
│  • 老板只动嘴，助理来干活                                        │
│  • 员工团队自动协作，产出完整作品                                 │
│  • 全程记录学习，越用越懂老板                                    │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 核心原则

| 原则 | 解释 |
|------|------|
| **唯一入口** | 只有老板助理对接老板，员工不直接对话 |
| **任务闭环** | 每个任务有启动→执行→汇报→确认→反馈完整流程 |
| **偏好积累** | 每次反馈都是训练数据，员工越用越聪明 |
| **掌控可见** | 老板随时可查看项目/员工/偏好一切状态 |

### 1.3 用户体验

```
老板: "帮我做个GPT-5的热点视频"

助理: "好的老板！正在为您创建项目...

📡 点子王正在为您分析热点...
━━━━━━━━━━━━━━━━━━━━━━━━━━━━"

1. GPT-5发布预告 (热度98)
2. 苹果折叠屏专利 (热度85)

建议方向：
• GPT-5科普解读

老板: "做GPT-5科普"

助理: "收到！任务安排：

### 2.2 模块职责

| 模块 | 职责 |
|------|------|
| 老板助理 | 唯一对话入口、意图理解、任务分发、进度汇报、偏好提取 |
| 点子王 | 热点分析、创意构思、方向建议 |
| 文案师 | 脚本撰写、旁白文案、标题封面 |
| 素材师 | 素材搜索、素材采集、素材整理 |

| 发布专家 | 多平台发布、数据追踪、定期汇报 |

## 三、核心流程

### 3.1 项目创建流程

```
1. 需求理解
   └── 提取：类型=热点视频，主题=GPT-5，目标=科普解读

2. 创建项目
   └── proj_xxx = { name, type, requirement, tasks: [] }

3. 任务拆解
   └── 点子王(热点) → 文案师(脚本) → 素材师(素材) → 剪辑师(成片)

4. 分配任务
   └── 按顺序召唤员工，传递任务要求

5. 汇报老板
   └── "请选择热点方向"
```

### 3.2 偏好学习流程

```
老板: "这个转场太生硬了，用点平滑过渡"

1. 反馈接收
   └── 对象: 剪辑师

2. 标签提取
   └── tags: ["转场=平滑过渡"]

3. 记录评语
   └── 保存原始评语到数据库

4. 更新偏好
   └── 剪辑师.preferences["转场"] = {
       value: "平滑过渡",
       confidence: 0.6
   }

5. 确认应用
   └── "收到！下次剪辑自动应用平滑转场"
```

// 助理自动提取并更新
员工档案.剪辑师.偏好.节奏 = "快节奏"
```

## 五、技术实现要点

### 5.1 Agent配置

- 主Agent: 老板助理（唯一对话入口）
- SubAgent: 6个员工Agent
- 使用sessions_spawn召唤员工

### 5.2 数据存储

- 项目: Memory中按项目ID存储
- 员工档案: Memory中按角色存储
- 偏好: 结构化存储，支持向量检索

### 5.3 Skill集成

已有能力封装为Skill：
- 文案→多分镜视频
- 热点→素材采编视频

## 六、与传统方案对比

| 维度 | 传统方案 | AI一人公司 |
|------|----------|------------|
| 交互方式 | 多Agent并列，用户自己协调 | 单一入口，AI协调 |
| 偏好记忆 | 无 | 持续学习，越用越懂 |
| 掌控感 | 低 | 高（项目/员工/偏好全可见） |
| 任务闭环 | 无 | 完整闭环 |
| 学习能力 | 无 | 反馈即学习 |

---

## 七、总结

AI一人公司的核心价值：

1. **让AI协调AI** - 老板助理作为唯一入口，协调其他AI员工
2. **让AI学习AI** - 偏好系统让员工越来越懂老板
3. **让老板有掌控** - 随时可查看项目进度、员工状态、自己的偏好
4. **让流程自动化** - 从需求到发布全自动完成
