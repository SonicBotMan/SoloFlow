## 二、流程执行

### 2.1 流程执行器

```javascript
class WorkflowExecutor {
  
  constructor(mcpServer) {
    this.mcp = mcpServer;
    this.workflows = {
      hot_video: hotVideoWorkflow,
      creative_video: creativeVideoWorkflow,
      product_video: productVideoWorkflow
    };
  }
  
  // 根据项目类型启动工作流
  async startWorkflow(projectId, projectType) {
    const workflow = this.workflows[projectType];
    if (!workflow) {
      throw new Error(`Unknown project type: ${projectType}`);
    }
    
    // 为每个步骤创建任务
    for (const step of workflow.steps) {
      const task = await this.mcp.create_task({
        project_id: projectId,
        employee_role: step.employee_role,
        name: step.name,
        description: step.description,
        requirement: step.additional_input || '',
        depends_on: step.order > 1 
          ? [`${projectId}_task_${step.order - 1}`] 
          : []
      });
    }
    
    // 开始执行第一步
    await this.executeNextStep(projectId, 1);
  }
  
  // 执行下一步
  async executeNextStep(projectId, stepOrder) {
    const project = await this.mcp.get_project({ project_id: projectId });
    const workflow = this.workflows[project.type];
    const step = workflow.steps.find(s => s.order === stepOrder);
    
    if (!step) {
      // 工作流完成
      await this.mcp.update_project_status({
        project_id: projectId,
        status: 'completed'
      });
      return;
    }
    
    // 检查前置依赖
    const tasks = await this.mcp.get_project_tasks({ project_id: projectId });
    const prevTask = tasks.find(t => t.employee_role === step.employee_role && t.status === 'completed');
    
    if (step.depends_on.length > 0 && !prevTask) {
      // 被阻塞，等待
      await this.mcp.update_task_status({
        task_id: `${projectId}_task_${stepOrder}`,
        status: 'blocked'
      });
      return;
    }
    
    // 召唤员工执行任务
    await this.spawnEmployee(step.employee_role, {
      project_id: projectId,
      task_id: `${projectId}_task_${stepOrder}`,
      step: step
    });
  }
  
  // 召唤员工
  async spawnEmployee(role, context) {
    // 使用 sessions_spawn 召唤对应的员工 Agent
    // ...
  }
}
```

## 三、状态流转

### 3.1 项目状态

```
draft → planning → executing → reviewing → completed
    ↓              ↓            ↓           ↓
  rejected      rejected    rejected   rejected
```

### 3.2 任务状态

```
pending → assigned → in_progress → waiting_review → completed
    ↓           ↓            ↓              ↓
  blocked    rejected     rejected       rejected
```

### 3.3 状态变更事件

```javascript
// 项目状态变更
const projectStatusFlow = {
  draft: {
    can_go_to: ['planning', 'rejected'],
    on_enter: '创建项目完成，开始任务拆解'
  },
  planning: {
    can_go_to: ['executing', 'rejected'],
    on_enter: '任务拆解完成，开始执行'
  },
  executing: {
    can_go_to: ['reviewing', 'rejected'],
    on_enter: '所有任务执行完成，等待验收'
  },
  reviewing: {
    can_go_to: ['executing', 'completed', 'rejected'],
    on_enter: '等待老板确认'
  },
  completed: {
    can_go_to: [],
    on_enter: '项目完成！'
  },
  rejected: {
    can_go_to: ['planning'],
    on_enter: '项目被驳回，需要重新规划'
  }
};
```

## 四、并行与串行

### 4.1 串行任务

默认情况下，任务按顺序执行：

```
Step 1 (点子王) → Step 2 (文案师) → Step 3 (素材师) → Step 4 (剪辑师)
     ↓
   等待
     ↓
   完成后才开始下一步
```

### 4.2 并行任务

某些情况下可以并行执行：

```javascript
{
  order: 3,
  employee_role: 'material',
  parallel_with: ['copywriter'],  // 可以和文案师并行
  name: '素材准备',
  // ...
}
```

```
Step 2 (文案师) ─┬─→ Step 3 (素材师) → Step 4 (剪辑师)
                 │
                 └─→ Step 3b (素材师) ← 并行执行
```

## 五、老板确认点

### 5.1 必须确认的点

| 节点 | 确认内容 | 老板选项 |
|------|----------|----------|
| 热点选择 | 选择要做的热点 | [确认] [换一个] |
| 创意方向 | 确认创意方向 | [确认] [修改] |
| 脚本确认 | 确认脚本内容 | [确认] [修改] |
| 成片验收 | 确认视频质量 | [确认] [打回] |
| 营销策略 | 确认发布策略 | [确认] [调整] |
